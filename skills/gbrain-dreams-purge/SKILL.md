# gbrain-dreams-purge

Continuous post-sync purge for dream content. Solves the gap left by the missing
upstream `.gbrainignore` feature (gbrain issue #1483). The on-disk dream files
are never deleted; this skill keeps them out of the search/embedding layer.

## Purpose

After every `gbrain sync`, delete any pages whose slug matches:

- `dreams` (the top-level `DREAMS.md` in each workspace)
- `memory/dreaming/%` (deep / light / rem buckets)
- `memory/.dreams/%` (legacy hidden dreams dir, one per-agent workspace only)

Cascade cleans `content_chunks`, `page_versions`, `page_links`, `page_tags`,
`raw_data`, etc. automatically.

## SQL contract

```sql
WITH del AS (
  DELETE FROM pages
  WHERE slug LIKE 'memory/dreaming/%'
     OR slug = 'dreams'
     OR slug LIKE 'memory/.dreams/%'
  RETURNING source_id
)
SELECT COUNT(*) AS deleted_count,
       string_agg(DISTINCT source_id, ',' ORDER BY source_id) AS sources
FROM del;
```

Idempotent — returns `0|` when nothing matches.

## Connection

```
postgresql://gbrain@localhost:5432/gbrain
```

Peer-auth (no password). Uses Homebrew Postgres 17 client at
`/opt/homebrew/opt/postgresql@17/bin/psql`.

## Exit-code contract

- `0` — success (whether 0 or N pages deleted)
- `1` — psql failed (DB down / client missing / role missing)

## Output contract

Per concise-output principle:

- 0 deleted → silent (no stdout)
- N>0 deleted → exactly one line to stdout:
  `dreams-purge: <N> pages deleted (sources: <comma-list>)`

Errors go to stderr.

## How it is wired

Chained at the end of `$HOME/.gbrain/postgres-sync-all.sh`, which is the
single canonical orchestrator for all gbrain sources. That script is invoked by
the `ai.gbrain.postgres-sync` LaunchAgent (every 300s, plus 08:00/09:00 calendar
ticks). One wiring point covers every source.

Any standalone per-agent curated-sync plist that only refreshes a curated mirror
under `gbrain-sources/<agent>/` does NOT call `gbrain sync` itself (postgres-sync
picks up the new commits). No purge call needed there.

## Standalone use

```bash
bash ${OPENCLAW_HOME}/skills/gbrain-dreams-purge/bin/run.sh
```

Safe to run anytime regardless of sync state.

## Failure modes

- If `postgres-sync-all.sh` is regenerated by a future `openclaw doctor --fix`
  or upstream gbrain installer, the trailing purge call may be stripped. The
  weekly invariants-check cron should grep for `gbrain-dreams-purge` in the
  sync wrapper and re-add it if missing.
- If gbrain ships native `.gbrainignore` (upstream #1483), this skill becomes
  redundant — replace with on-disk ignore file + remove the chained call.
