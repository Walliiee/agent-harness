# secrets/

This directory deliberately holds **no live secret values**. It contains:

- This README — the enumeration of every secret the system needs and how to re-issue or re-establish it.
- (`.env` and `.local` files are gitignored — if you create them locally, they will not be committed.)

The harness is designed around the assumption that **all secrets are reissuable** from their service dashboards or by re-running their pairing/registration flows. No password manager is required to restore from this repo, though one is strongly recommended for ongoing operation.

## Restore order

When following `../../docs/disaster-recovery.md`, secrets are restored in this order:

1. **GitHub PAT** — needed first to clone any private repos.
2. **API keys** (Anthropic, Gemini, Tavily, Brave) — needed before any agent can run.
3. **GitHub PAT for MCP** — needed before the MCP GitHub server can authenticate.
4. **gbrain OAuth client credentials** — auto-registered via DCR; no manual step.
5. **gbrain access tokens** (`GBRAIN_TOKEN_*`) — auto-minted by `bin/gbrain-token-refresh.sh` after step 4.
6. **Telegram bot tokens + pairing** — re-create bots in BotFather; pair via `device-pair` plugin.
7. **Gateway auth token** — auto-generated by `openclaw doctor --fix` on first run.
8. **OpenAI/Codex auth profile** — re-run `openclaw auth` flow.

## Inline placeholders in `config/openclaw.json.template`

These are the `${VAR}` references in the openclaw.json template. Each must be replaced with a real value before openclaw will start cleanly.

### Provider API keys

| Placeholder | Service | Re-issue from |
|---|---|---|
| `${ANTHROPIC_API_KEY}` | Anthropic API | https://console.anthropic.com/settings/keys |
| `${GEMINI_API_KEY}` | Google AI Studio | https://aistudio.google.com/apikey |
| `${TAVILY_API_KEY}` | Tavily Search | https://app.tavily.com (Account → API Keys) |
| `${BRAVE_API_KEY}` | Brave Search API | https://api-dashboard.search.brave.com |
| `${GITHUB_PERSONAL_ACCESS_TOKEN}` | GitHub MCP server | https://github.com/settings/tokens — needs `repo` + `read:org` scopes |

### gbrain access tokens (auto-minted)

| Placeholder | Notes |
|---|---|
| `${GBRAIN_TOKEN_MAIN}` | Auto-refreshed by `bin/gbrain-token-refresh.sh` via OAuth client-credentials flow |
| `${GBRAIN_TOKEN_DEV}` | Same |

(Add one row per agent that talks to gbrain.)

**Restore flow:** ensure gbrain is running (port 8182), then run `~/.openclaw/bin/gbrain-token-refresh.sh` once. The script handles Dynamic Client Registration if no `~/.openclaw/credentials/gbrain-oauth.json` exists, then mints tokens for all agents and patches them into `openclaw.json` via `openclaw config patch`.

### Template / deployment variables

| Placeholder | Source |
|---|---|
| `${OPENCLAW_HOME}` | Your home dir (or install root). Expanded by `bootstrap.sh` when it renders the template; defaults to `$HOME`. |
| `${GH_ORG}` | Your GitHub org/user. Set as an env var before running `restore-workspaces.sh`. |
| `${HARNESS_TG_CHAT_ID}` | Your Telegram chat/owner ID — the account allowed to talk to the bots. |

### OpenClaw internal

| Placeholder | Source |
|---|---|
| `${OPENAI_CODEX_PROFILE_ID}` | Auto-assigned during `openclaw auth` Codex flow. Replace the placeholder key in `auth.profiles` after first successful sign-in. |
| `${OPENAI_CODEX_EMAIL}` | The email used for Codex sign-in. |

The hooks token (`<REDACTED_TOKEN>` in the template) is auto-generated/rotated by openclaw on first start.

## Secrets referenced by openclaw.json but stored elsewhere

`openclaw.json` references several secrets indirectly via `{source:file, id:/NAME}` pointers. The actual values live in the file pointed at by `secrets.providers.local.path` (`${OPENCLAW_HOME}/secrets.json` in the template).

### Local secrets file contents

| Secret ID | What it is | Re-issue from |
|---|---|---|
| `/GATEWAY_AUTH_TOKEN` | Random token gating local gateway access | `openclaw doctor --fix` regenerates |
| `/TELEGRAM_TOKEN_MAIN` | Bot token for the main agent | Telegram → @BotFather → `/mybots` → revoke + regenerate |
| `/TELEGRAM_TOKEN_DEV` | Bot token for the dev agent | Same |

(Add one row per agent with a Telegram channel.)

The file format is JSON keyed by these IDs. Owner-only permissions (`chmod 600`).

## Files in `~/.openclaw/credentials/` (NOT in this repo)

Three files / directories of credentials live locally and are NOT included in the DR repo:

| Path | Contents | Restore |
|---|---|---|
| `~/.openclaw/credentials/gbrain-oauth.json` | OAuth client_id + client_secret per agent | Auto-recreated by `bin/gbrain-token-refresh.sh` (the `register_client` function does DCR against gbrain's `/register` endpoint) |
| `~/.openclaw/credentials/telegram-pairing.json` | Telegram device-pair state | Re-pair each bot via the `device-pair` plugin after restoring tokens |
| `~/.openclaw/credentials/auth-profiles/` | openclaw account auth artifacts | Re-run `openclaw auth` for each provider in `auth.profiles` |
